## Key Design Decisions

### Why 3 Layers?
- **Layer 1:** Catches misclicks and provides information
- **Layer 2:** Provides "cooling off" period through typing requirement
- **Layer 3:** Final chance to review all details before permanent action

### Why Typed Confirmation?
- Forces user to slow down and read event name
- Cannot be completed by muscle memory or rapid clicking
- Provides clear cognitive engagement with the action
- Industry standard for destructive actions (GitHub, AWS, etc.)

### Why Soft Delete?
- Provides safety net for accidental deletions
- Allows recovery without database backups
- Maintains audit trail of deletions
- Can be hard-deleted later by scheduled job

### Why Block Currently Selected Event?
- Prevents context loss bugs
- Avoids race conditions in UI state
- Forces user to consciously switch context
- Clearer user experience (switch then delete)

### Why Danger Zone Section?
- Visually separates destructive actions
- Users must scroll to reach it (intentional)
- Red color scheme signals danger
- Industry standard pattern (GitHub, Stripe, etc.)
